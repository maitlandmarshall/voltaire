# Voltaire’s Codex

The `Codex/` directory is Voltaire’s *stateful* memory store: a structured reference of what Voltaire knows (and what the party knows), maintained across sessions.

Quick entry point: `Codex/Index.md`.

## What belongs here

- **Items**: artifacts, magic items, documents, gear, currencies, etc.
- **Lore**: deities, planes, metaphysics, historical notes, recurring concepts.
- **NPCs**: named characters, monsters-with-names, patrons, rivals, etc.
- **Powers**: Voltaire’s unique powers / boons / homebrew effects.
- **Places**: locations, dungeons, cities, planes, strongholds.
- **Factions**: organizations, cults, houses, churches, armies.

## Knowledge tiers (important)

When writing/updating Codex pages, prefer to separate:
- **Party Knowledge**: facts revealed in-play to the party.
- **Voltaire-Only Knowledge**: things only Voltaire knows (including anything derived from `Module/` unless it became known in-play).
- **Speculation / Theories**: label clearly as non-canon until confirmed.

## Folder map

- `Codex/Items/`
- `Codex/Lore/`
- `Codex/Characters/`
- `Codex/Powers/`
- `Codex/Places/`
- `Codex/Factions/`

## Conventions

- Prefer one concept per file.
- Keep filenames human-readable (spaces are OK; match existing style).
- Use wikilinks (double-bracket links) for cross-references; use path links only when needed.
